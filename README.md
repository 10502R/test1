# Contributing Guide

## 브랜치
- main 브랜치는 배포용으로 직접 푸시 금지
- dev 브랜치에 기능 브랜치(feature/*) 머지
- 새 기능은 반드시 feature 브랜치에서 작업

## PR (Pull Request)
1. feature 브랜치 → dev 브랜치로 PR 생성
2. 최소 1명 이상 리뷰 후 머지
3. main 브랜치는 dev에서만 머지
